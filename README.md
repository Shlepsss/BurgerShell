# Burger Shell

Это DOS оболочка, похожая на Windows 1.0, является учебным проектом.
Связи между этой оболочкой и бургерами - 0%
Это не ОС под чистый BIOS, а только оболочка под DOS (real mode), которая использует:
- графику через BIOS/VGA Mode `13h`
- мышь через драйвер CuteMouse `INT 33h`
- файловую систему FAT через DOS (обычный `fopen/fread/fwrite`)
- Простые самописные библиотеки
- Очень облегченный FreeDOS

## Преимущества этой системы
- Может поместиться буквально на любой загрузочный носитель, так как вес системы 174 килобайта
- Библиотеки очень просты в понимании и обращаются с BIOS как с родным братом
- Использует популярный WATCOM C
- Нету НИ одного файла на ассемблере, поймет даже дитё

## Сборка

- `build.bat` собирает `SHELL.COM` и использует сборщик и линкер WATCOM (wcl)
- Необходим WATCOM, поддержка x16 small, а также нужно чтобы WATCOM был прописан в PATH системы

## Запуск пустого .com

- В DOSbox'е или любой другой среде: `shell.com`

### Образ дискеты

- Записать на дискету или USB флешку
- Если с дискеты на старом ПК то ничего не нужно дополнительного
- Если с USB на современном ПК, убедитесь что в UEFI включен режим CSM для совместимость с BIOS.

## Управление:
- верхняя панель: `Files`, `Calc`, `About`, `Pwr`, `Game`, `Exit`
- в `Files`: клик по файлу `.TXT`, затем `Enter` чтобы открыть
  
  ``
  Чтобы создать файл нужно перейти в DOS режим и прописать echo 1 >название.txt
  ``
  
- в редакторе текста: печать ASCII, `Backspace`, `Enter`, `Ctrl+S` - сохранить, `ESC` - закрыть
- в `Calc`: набрать пример на клавиатуре и нажать `enter`
- в `Game` (змейка): `WASD` - управление змейкой, `ESC` - закрыть

## Содержание дискеты
- Ядро DOS kernel.sys
- Файл-конфигуратор FreeDOS fdconfig.sys
- Автозагрузчик (а-ля AUTOEXEC.BAT) fdauto.bat
- Интерфейс DOS COMMAND.COM
- Драйвер для мышки CuteMouse MOUSE.COM
- Сама оболочка SHELL.COM
